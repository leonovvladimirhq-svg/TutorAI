# CLOUD.md — TutorAI в облаке

Документ описывает, **что это за проект, как он устроен, какая облачная инфраструктура
нужна и как его развернуть** в Yandex Cloud. Читается «с нуля» — без знания истории разработки.

---

## 1. Что это и зачем

**TutorAI («Наставник AI»)** — Telegram-бот, персональный ИИ-наставник студента для ОП
«Интегрированные коммуникации» ШК ВШЭ. Помогает студенту:

- собрать **учебно-карьерный профиль** в коучинговом диалоге (текст или голос);
- сформулировать **цели по методике SMART** (бот оценивает цель студента по SMART и коучит,
  помогая довести её до сильной формулировки, а не выдаёт готовую цель);
- видеть и редактировать свой профиль;
- (в дальнейшем) получать напоминания о дедлайнах, рекомендации курсов, аналитику для наставника.

**Роли и доступ.** Доступ к боту — по роли, привязанной к Telegram ID: **Студент**,
**Наставник**, **Академический руководитель**. Роли назначает академический руководитель
через **веб-панель** (отдельный сервис `web`). Пока роль не назначена — бот блокирует
пользователя и показывает его Telegram ID для передачи руководителю. Паролей и выбора
профиля больше нет.

Зачем: на курсе 2 наставника на ~75 студентов — глубокой персональной работы на всех не хватает.
Бот берёт на себя рутину (профайлинг, постановка целей, аналитика), не заменяя наставника.
Бот — `@tutor_hse_ai_bot`. Это прототип; цель — рабочая, расширяемая архитектура.

## 2. Архитектура

```
Студент ──Telegram (текст/голос)──► Бот (aiogram, long-polling) ┐
                                       │                          │
Академ. руководитель ──HTTP──► Веб-панель (FastAPI, порт 8080) ───┤ общая БД
                                       │                          │
                  ┌────────────────────┼─────────────────────┐   │
                  ▼                    ▼                     ▼    ▼
            PostgreSQL          Yandex AI Studio        Yandex SpeechKit
        (роли, профиль,          (Qwen — диалог,          (распознавание
         цели, история,           извлечение фактов,       голоса → текст)
         события)                 оценка SMART-целей)
```

- **Канал** — Telegram (long-polling, домен/webhook не нужны).
- **Роли** — реестр `app_user` (Telegram ID → роль); управляется через веб-панель.
- **Веб-панель** (`app/web`, FastAPI + Jinja2) — вход по логину/паролю, назначение ролей
  академическим руководителем; переиспользует общую БД. Три контейнера: `migrate`, `bot`, `web`.
- **Ядро** — структурированный профиль студента с провенансом (у каждого факта — источник
  и статус подтверждения). В профиль попадает только то, что студент подтвердил.
- **LLM** — Qwen в Yandex AI Studio (OpenAI-совместимый API, авторизация IAM-токеном).
- **STT** — Yandex SpeechKit (OggOpus из Telegram → текст).
- **Хранилище** — PostgreSQL (в прототипе — контейнер; для прода можно Managed PostgreSQL).
- **Метрики** — таблица событий + команда `/stats` (доля SMART-целей, заполненность профиля,
  вовлечённость).

Подробнее о коде — в [README.md](README.md).

## 4. Облачная инфраструктура (Yandex Cloud)

Что используется в каталоге Yandex Cloud:

1. **AI Studio** — инференс модели Qwen (Qwen3 30B-A3B). Оплата по токенам.
2. **SpeechKit** — распознавание речи. Оплата по секундам аудио.
3. **Compute Cloud (VM)** — одна виртуальная машина, на которой в Docker работают бот и Postgres.
4. (Опционально для прода) **Managed PostgreSQL** вместо контейнера.

GPU не нужен: модель крутится в AI Studio, на VM — только лёгкие бот и БД.

### 4.1. Конфигурация VM и стоимость

Рекомендуемая (экономичная) конфигурация:

| Параметр | Значение | Зачем |
|----------|----------|-------|
| Платформа | standard-v3 | актуальная серия |
| vCPU | 2 ядра, **гарантированная доля 50%** | бот почти всё время простаивает → платим вдвое меньше за vCPU |
| RAM | 4 ГБ | хватает боту + Postgres (лимит заказчика — ≤6 ГБ) |
| Диск | network-HDD, 20 ГБ | дешевле SSD, для прототипа достаточно |
| ОС | Ubuntu 22.04 LTS | |
| Прерываемая | нет | бот должен работать постоянно |
| Публичный IP | да (NAT) | для исходящих запросов и polling |

**Оценка стоимости:** такая VM на on-demand ориентировочно укладывается в **~1000–1500 ₽/мес**,
что ниже целевого потолка (1500–2000 ₽/мес). Точную сумму даёт
[калькулятор Yandex Cloud](https://yandex.cloud/ru/prices). Снизить ещё:

- **Committed-use (резервирование ресурсов на срок)** — скидка за обязательство, как раз тот
  механизм «гарантированного резервирования», о котором речь: при коммите на 1–3 мес/года
  стоимость VM падает ощутимо.
- **network-HDD** вместо SSD (уже учтено).
- Инференс AI Studio и SpeechKit оплачиваются отдельно по факту; на объёме пилота
  (десятки студентов) это малые суммы.

После того как пришлёте **FOLDER_ID** (и роли выданы) — я за один проход:

## 6. Развёртывание

Полные шаги — в [DEPLOY.md](DEPLOY.md). Кратко:

- **Создание VM:** `FOLDER_ID=<folder> SSH_KEY=~/.ssh/id_rsa.pub bash scripts/deploy_yc.sh`
  (создаёт экономичную VM из §4.1, ставит Docker, клонирует репозиторий).
- **Запуск:** залить `.env` и `secrets/sa-key.json` на VM → `docker compose up -d --build`
  (поднимутся `migrate`, `bot`, `web`).
- Миграции БД применяются автоматически одноразовым сервисом `migrate`.
- Проверка модели до запуска: `docker compose run --rm bot python -m scripts.smoke_llm`.
- **Веб-панель ролей:** `http://<IP>:8080` (логин `academ`/`ABCD` по умолчанию). На VM/сервере
  откройте входящий TCP-порт **8080** (security-group / firewall).

Доступ в бота — по роли из веб-панели; первого руководителя можно задать `BOOTSTRAP_DIRECTOR_TG` в `.env`.

## 7. Секреты и безопасность

- Секреты (`.env`, `secrets/sa-key.json`) **не хранятся в репозитории** (`.gitignore`).
- Токен бота и ключ сервисного аккаунта передаются на VM отдельно (scp), не через git.
- Рекомендация: после настройки **перевыпустить** токен бота (@BotFather) и ключ SA —
  они ранее передавались в переписке.
- **Веб-панель:** логин/пароль (`WEB_ADMIN_USER`/`WEB_ADMIN_PASSWORD`, по умолчанию
  `academ`/`ABCD`) и секрет сессии (`WEB_SESSION_SECRET`) задаются в `.env`. Пароль `ABCD`
  и доступ по HTTP — прототип; для прода задайте сильный пароль, длинный секрет и HTTPS.
- Обработка ПДн: при первом запуске бот показывает экран согласия; данные не покидают
  российский контур (Yandex Cloud, 152-ФЗ).

### Управление на VM

```bash
ssh yc-user@51.250.11.57
cd /opt/tutorai
sudo docker compose ps               # статусы (db / migrate / bot / web)
sudo docker compose logs -f bot      # логи бота
sudo docker compose logs -f web      # логи веб-панели
sudo docker compose restart bot      # перезапуск
git pull && sudo docker compose up -d --build   # обновление из main
```

Автозапуск после перезагрузки VM обеспечен (`restart: unless-stopped` + включённый docker).
Порты 22 и 8080 доступны (на сетевом интерфейсе VM не задана security-group → трафик открыт).
